### âœ… Validators
Validators are functions that **check input values** against certain rules or constraints.  
If validation fails, errors are added to the `validationResult(req)`.

**Common validator functions:**
- `isEmail()` â†’ checks if value is a valid email format  
- `isLength({ min, max })` â†’ enforces string length constraints  
- `isInt({ min, max })` â†’ ensures value is an integer in range  
- `isFloat({ min, max })` â†’ ensures value is a float in range  
- `isBoolean()` â†’ checks if value is boolean  
- `isURL()` â†’ validates URL format  
- `isAlphanumeric()` â†’ letters and numbers only  
- `isEmpty()` / `notEmpty()` â†’ checks emptiness  
- `matches(regex)` â†’ matches a regex pattern

### ğŸ§¹ Sanitizers
Sanitizers **clean or transform input values** to a desired form.  
They help standardize input before validation.

**Common sanitizer functions:**
- `trim()` â†’ removes whitespace from both ends  
- `escape()` â†’ replaces `<`, `>`, `&`, `'`, `"` with HTML entities  
- `normalizeEmail()` â†’ standardizes email format  
- `toInt()` â†’ converts input to integer  
- `toFloat()` â†’ converts input to float  
- `toBoolean()` â†’ converts to boolean  
- `customSanitizer(fn)` â†’ apply a custom sanitization function

### âœï¸ Modifiers
Modifiers are functions that **adjust the flow or error handling** of validation chains.

**Common modifier functions:**
- `withMessage('msg')` â†’ attach custom error messages  
- `optional()` â†’ makes a field optional (skip validation if empty)  
- `bail()` â†’ stop running further validators if current one fails  
- `custom(fn)` â†’ run custom validation logic  
- `if(condition)` â†’ run validator only if condition passes  

---

### âš–ï¸ Best Practice: Order Matters
- **Sanitize first â†’ then validate.**  
  Example: `body('email').trim().normalizeEmail().isEmail()`  
- If you validate before sanitizing, you might record errors that the sanitizer would have fixed.  
- Use `.bail()` when you *want to stop* further checks after a failure.
